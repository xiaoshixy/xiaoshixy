项目管理基础和项目立项

# 项目

# 五个特点

点击查看

🔸临时性

  • 有明确的起点和终点

🔸独特性

  • 过程独特、结果独特

🔸渐进明细性

  • 由粗到细逐步完善

🔸资源约束性

  • 资源成本是一个约束条件

🔸目的性

  • 三约束----进度、成本、质量
  • 四要素----进度、成本、质量、范围

# 信息系统项目的特点

目标不明确、 需求变化频繁、 智力密集型

# 项目经理

# 责任

点击查看

⭐️任务需求、团队需求、个人需求

# 能力

点击查看

⭐️知识能力、实践能力、个人能力

# ⭐如何做好一个项目经理

点击查看
  1. 真正理解项目经理的角色、重视项目团队的管理
  2. 真正理解一把手工程，注重用户参与
  3. 必须承担管理者和领导者的双重角色

# ⭐组织结构

点击查看

💙职能型

  • 项目经理几乎无权
  • 兼职项目经理（联络员）
  • 职业路径清晰
  • 横向联系薄弱

💙项目型

  • 项目经理几乎全权
  • 利于指挥、沟通方便
  • 重复配置、不利于知识共享、无家可归

💙矩阵型

资源利用率高、有利于跨部门协调

多头领导；管理难度大；资源争夺

  • 弱矩阵

    • 兼职项目经理（协调员）
    • 职权： 小
    • 项目经理<职能经理

  • 平衡矩阵

    • 第一次出现全职项目经理
    • 职权： 小~中
    • 项目经理=职能经理

  • 强矩阵

    • 有项目经理的经理
    • 职权： 中~高
    • 项目经理>职能经理

# PMO

# 三种类型

点击查看

❤️支持型——对项目控制程度很低

❤️控制型——对项目控制程度中等

❤️指令型——对项目控制程度很高

# 作用【案例】

点击查看

🔸管理

管理“共享资源”，识别和制定“最佳实践”和“标准”

🔸监督

通过“项目审计”， 监督对“标准”的遵守程度

🔸指导培训

制定和管理政策、程序、模板，提供指导和培训

🔸协调

协调“跨项目”的沟通

# 项目生命周期

# 生命周期

阶段：顺序关系”或“交叠关系”

  1. 项目生命周期

    • 时间维度划分

      具体

      项目从启动到收尾所经历的一系列阶段，阶段名称和数量视具体项目而定

  2. 项目通用生命周期

    • 4个通用阶段

    启动项目；

    组织与准备；

    执行项目工作；

    结束项目

    • 特征
    点击查看

    成本与人力投入

    开始缓慢增加，执行最高，结束迅速回落

    风险与不确定性、干系人的影响力、变更的数量

    项目开始最大，后续逐步降低

    变更的代价、风险的影响

    开始较小，后续显著提高

  3. 产品生命周期

    具体

    从项目开始到项目结束再到项目产品运行生命终止（退出市场）的全过程

# 项目管理过程组

启动，规划，执行，监控，收尾

# ⭐项目生命周期模型

# 瀑布型

点击查看
  • 特点：
    • 计划驱动、顺序执行
    • 常用结构化方法
  • 适用：
    • 需求明确、有行业基础

# 螺旋型

点击查看
  • 特点：
    • 风险驱动、过程迭代
    • 瀑布+快速原型
  • 适用:
    • 大型复杂、 高风险

# 迭代型

点击查看

  • 特点：

    • 重复循环、完善迭代

  • 适用:

    • 不能完整定义所有需求、计划多期开发

# 增量型

点击查看

  • 特点：

    • 渐进增加、功能迭代

  • 适用:

    • 已有产品升级或新版本开发

# 敏捷型

点击查看

  • 特点：

    • 变更驱动、小步快跑、每次交付最有价值的成果、频繁交付

  • 适用：

    • 需求多变

# 原型化

点击查看
  • 特点：
    • 动态响应、逐步纳入、快速开发、反复修改
  • 适用：
    • 需求定义不清，结构化不高

# 统一过程模型RUP

点击查看

  • 特点：

    • 用例驱动、面向对象
    • UML方法

  • 适用：

    • 大型软件项目和开发团队

# 喷泉模型

点击查看

  • 特点：

    • 对象驱动、面向对象
    • 无间隙、复用

  • 适用：

    • 面向对象

# V模型

点击查看

  • 特点：

    • 细化瀑布模型测试部分

  • 适用：

    • 需求明确、需求变更不频繁的情况

  • 输出

    机会研究报告

  • 作用 初步判断项目是否有生命力，能否盈利

  • 输出

    初步可行性研究报告

# ⭐详细可行性研究

必须要有

  • 作用

    详细论证，多方案比较选择

  • 输出

    可行性研究报告

  • 投资估算法
  • 增量净效益法
  • 经济评价法
  • 市场预测法

# ⭐内容

背景/意义/必要性；需求；资源设施

范围技术；进度；资金

人力；效益；合作

点击查看

🔸概述

🔸资源设施分析

🔸初步技术方案

🔸进度计划建议

🔸投资估算和资金筹措

🔸人力资源培训

🔸经济和社会效益分析

🔸合作/协作方式建议

# ⭐可研报告中的开发总成本

  1. 经营成本

    • 研发成本
    • 行政管理费
    • 销售与分销费用

  2. 非经营成本

    • 财务费用和折旧

# ⭐可行性

点击查看

🔸技术可行性

技术能力、开发风险、人力资源有效性、物资可用性

🔸经济可行性

支出分析—一次性和非一次性；收益分析—直接和间接；收益投资比和投资回收期分析、 敏感性分析

🔸运行环境可行性

评估系统顺利运行所需的环境及建立环境所需的工作

🔸其他方面可行性

法律可行性、社会可行性、财务可行性、组织可行性、 投资必要性、 风险因素及对策—市场风险等
