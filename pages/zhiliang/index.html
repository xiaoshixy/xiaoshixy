项目质量管理

  • 戴明
    • PDCA环
    • 质量管理14条原则（预防胜于检测，质量成本85%都是管理问题，持续改进）
  • 朱兰
    • 质量螺旋
    • 质量是适合使用，质量与等级的区别
    • 质量管理三部曲
  • 克鲁斯比
    • 零缺陷，质量是符合要求
    • 第一次就把事做对
    • 质量是免费的
  • 石川
    • 鱼骨图（因果图、石川图）
    • 质量圈
    • 质量七工具的使用
  • 田口玄一
    • 质量损失函数
    • 实验设计方法
    • 稳健设计方法

# 质量

定义

作为实现的性能或成果，一系列内在特性满足要求的程度

# 等级

作为设计意图，对用途相同但技术特性不同的可交付成果的级别分类

# ⭐️ISO 9000 质量管理8项基本原则

领导全员用管理事实的方法，对顾客和供方进行过程改进

点击查看
  • 领导作用——全员参与
  • 管理的系统方法—— 基于事实的决策方法
  • 以顾客为中心——与供方互利
  • 过程方法——持续改进

# 规划质量管理

定义

识别 项目及其可交付成果的 质量要求和标准

书面描述项目将 如何证明符合质量要求和标准

# 输入

点击查看

干系 人登记册——识别对质量重视或有影响的干系人

风险登记册——包含影响质量的各种威胁和机会信息

需求文件——包括质量需求

组织过程资产——执行组织的质量政策是高级管理层所推崇的

# 工具

点击查看

  • 成本效益分析（比较可能成本与预期效益）

  • 质量成本COQ

    质量方面花的所有钱

    • 一致性成本（为了防止失败）
      • 预防成本（培训，设备）
      • 评估成本（检查，测试）
    • 非一致性成本（为了处理失败）
      • 内部失败成本（返工，废品）
      • 外部失败成本（责任，保修）

  • 标杆对照

    识别最佳实践 形成改进意见 ，并 为绩效考核提供依据

  • 实验设计DOE

    一种统计 方法，用来识别 哪些因素 会对 特定变量 产生影响

  • 力场分析

    通过找出各种有利于解决问题的力量和因素，加强积极因素，排除消极因素

  • 老七工具【因流核帕直控散】

    因果图 追溯问题来源，根本原因

    流程图 SIPOC模型，寻出错环节

    核查表 收集数据

    帕累托图 主要原因 80/20法则

    直方图 趋势分布

    控制图 判过程稳定

    • 失控的判断：一点原则，七点原则

    散点图 两个变量

# 输出

点击查看
  • 质量管理计划（如何做）
  • 过程改进积极（识增值）
  • 质量测量指标（定指标）
  • 质量核对单（给QC)

# 质量管理计划内容

点击查看
  1. 质量政策、质量目标；
  2. 质量标准；
  3. 质量角色与职责；
  4. 质量工具；
  5. 需要质量审查的项目可交付成果和过程；
  6. 为项目规划的质量控制和质量管理活动

# 实施质量保证

定义

审计质量要求和质量控制测量结果， 确保 采用 合理的质量标准和操作性定义的过程

# 做的几件事

点击查看
  1. 按质量管理计划和质量测量指标做出合格的质量
  2. 识别 最佳实践、违规做法、差距及不足
  3. 按过程改进计划， 改进生产过程，消除非增值活动
  4. 对照实际质量绩效，考察质量标准和可操作定义的合理性， 提出必要的变更请求
  5. 确认 已批准的变更请求的实施情况 （变更做没做）
  6. 提高主要项目干系人对项目将要达到质量要求的信心

# 输入

点击查看

质量管理计划

过程改进计划

质量测量指标

质量控制测量结果

# ⭐️工具

点击查看

  • 新七工具【矩树优活亲过关】

    亲和图 分类,有助于WBS制定

    树形图 层次分解结果

    过程决策程序图 （PDPC） 理解一个目标 与 达成此目标的步骤之间的关系

    矩阵图 变量之间相关性的强弱

    优先矩阵 加权、打分、排序

    关联图 有助于在包含相互交叉逻辑关系的 中等复杂情形 中 创新性地解决问题

    活动网络图 连同项目进度计划编制方法一起使用

  • ⭐️质量审计【过程】

    1. 识别最佳实践、违规做法、差距及不足
    2. 分享类似项目的良好实践
    3. 协助过程改进、提高生产效率
    4. 积累经验教训
    5. 确认已批准的变更请求的实施情况

  • 过程分析(根本原因分析)

# 输出

点击查看

变更请求

# 控制质量

定义

监督并记录质量活动执行结果，以便评估绩效，并推荐必要的变更过程

# 做的几件事

点击查看
  1. 用质量核对单 检查项目管理工作的质量和可交付成果质量 ，并记录成质量控制测量结果
  2. 整理出工作绩效信息 ，并 提出变更请求
  3. 检查 已批准的 变更请求 是否已得到合理 实施 （变更做的对不对)

# 输入

点击查看

质量测量指标、质量核对单

批准的变更请求——>确认的变更

可交付成果——>核实的可交付成果

# 工具

点击查看

  • 统计抽样

    • 属性抽样 合格还是不合格
    • 变量抽样 合格的程度

  • 检查

# 输出

点击查看
  • 质量控制测量结果
  • 核实的可交付成果
  • 确认的变更
