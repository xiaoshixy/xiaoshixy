# 信息

# 概念

💠是一种客观事物，能够用来消除不确定性

# 信息的特征

点击查看

🔸客观性——主观信息必然转换为客观信息

🔸依附性——不能脱离物质独立存在

🔸传递性——时间上传递是存储；空间传递是转移或扩散

🔸转换性——将信息转化为物质或能量

口诀

安全靠精整及经验

点击查看

🔹安全性——可被非授权访问

🔹可靠性——可信任

🔹精确性——精准程度

🔹完整性——全面程度

🔹及时性——发生间隔长短

🔹经济性——成本在可接受范围

🔹可验证性——证伪或证实

# 信息系统4种开发方法

# 结构化方法

点击查看

🔹精髓：自顶向下、逐步求精、模块化设计

🔹优点：逐层分解、逐步求精、强调完整顺序

🔹缺点：开发周期长 ，难以适应需求变化

🔹适用：数据处理领域

🔹工具：DFD、E-R、数据字典、判定树

# 面向对象方法

点击查看

🔹优点：复用性好、缩短开发周期

🔹缺点：必须依靠一定的OO技术

🔹适用：普遍适用各类系统

🔹工具：UML

# 基本概念

类 （事物的抽象，包含 属性、操作 ）

对象 （类的实例化)

继承 （子类继承父类所有功能 ，并进行扩展 ）

封装 （隐藏实现细节、对外公开接口)

接口 （一种规范 ，用于和外部实体消息通信 ）

多态(一个接口，多种实现)

# 原型化方法

点击查看

🔹优点：开发周期短、综合开发效益高、利于移交和维护

🔹缺点：开发环境要求高

🔹适用：需求不明确，分析层面难度大

  • 水平原型：未实现功能，用于界面，应用需求
  • 垂直原型：实现部分功能，用于复杂算法，应用设计
  • 抛弃型原型：达成目的，原型本身被抛弃
  • 演化式原型：逐步演化最终系统
