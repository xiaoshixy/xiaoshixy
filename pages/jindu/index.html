项目进度管理

# 规划进度管理

定义

为规划、编制、管理、执行和控制项目进度而制定政策、程序和文档

# 输入

点击查看

项目章程——>总体里程碑计划和项目审批要求

项目管理计划——>开发方法有助于定义进度计划方法

# 工具

  • 分析技术
  • 会议

# 输出

点击查看
  • 进度管理计划

进度管理计划无进度，有准确度，计量单位，控制临界值，绩效测量规则

# 定义活动

定义

识别和记录为完成项目可交付成果而需采取的具体行动

# 输入

点击查看

进度管理计划——>滚动式规划持续时间以及管理工作详细程度

范围基准——>考虑范围基准中的WBS、可交付成果、制约因素、假设条件

# 工具

  • 分解
  • 滚动式规划

# 输出

点击查看
  • 活动清单/活动属性

包含项目所需的全部进度活动的综合清单

  • 里程碑清单

只代表一个重要事件，持续时间为0

# 排列活动顺序

定义

识别和记录项目活动之间的关系

# 输入

点击查看

活动清单

活动属性

里程碑清单

项目范围说明书——>项目的制约因素、假设条件

# 工具

点击查看

  • 紧前关系绘图法PDM

    • 四种逻辑关系
      • FS、 SS、 FF、 SF

  • 确定依赖关系

    • 强制性依赖关系
      • 法律或合同要求的或工作的内在性质决定的依赖关系
      • 项目团队不能违反
    • 选择性依赖关系
      • 基于最佳实践建立的、或基于项目的某些特殊性质而采用的依赖关系
      • 项目团队可自由选择
      • 可供打算快速跟进时审查
    • 外部依赖关系
      • 项目活动与非项目活动之间的依赖关系
      • 项目团队不可控
    • 内部依赖关系
      • 是项目活动之间的紧前关系
      • 项目团队可控

  • 提前量与滞后量

    • 提前量
      • 相对于紧前活动，紧后活动可以提前的时间量
    • 滞后量
      • 相对于紧前活动，紧后活动必须推迟的时间量

  • 箭线绘图法

    • ADM、双代号法、 AOA

# 提前量与滞后量练习

点击查看

⭐️SS、SF

F结束时间——>当天晚上

开始时间S——>后一天的早上

FF+2:

A：本来是第五天晚上结束，+2是往后推两天，第7天晚上结束

B：第7天晚上结束

SS+2:

A：本来是第一天早上开始，+2是往后推两天，第3天早上开始

B：第3天早上开始

SF+2:

A：本来是第一天早上开始，+2是往后推两天，第3天早上开始

B：第3天结束

FS-2:

A：本来是第5天晚上结束，-2是往前推两天，第3天晚上结束

B：不能第3天晚上开始，只能第4天早上开始

FF-2:

A：本来是第5天晚上结束，-2是往前推两天，第3天晚上结束

B：第3天晚上结束

SS-2:

A：本来是第3天早上开始，-2往前推两天，第1天早上开始

B：第1天早上开始

SF-2:

A：本来是第3天早上开始，-2往前推两天，第1天早上开始

B：第1天早上结束不可以，只能是第0天晚上结束

# 输出

  • 项目进度网络图

表示项目活动之间的逻辑关系图形

# 估算活动资源

定义

估算执行各项目活动所需材料、人员、设备或用品的种类和数量

# 输入

点击查看

资源日历——>判断时间上是否可用

活动成本估算——>估算出的资源所需成本与活动成本估算比较，判断成本合理性

# 工具

  • 备选方案分析
  • 发布的估算数据
  • 自下而上估算

# 输出

点击查看
  • 活动资源需求

明确工作包每个活动所需资源类型和数量

  • 资源分解结构

资源类别和类型的层级展现

# 估算活动持续时间

定义

根据资源估算的结果，估算完成单项活动所需工作时段数

# 输入

点击查看

活动资源需求

资源日历

项目范围说明书

# 工具

点击查看

  • 类比估算

  • 参数估算

    • 准确度取决于参数模型的成熟度和基础数据的可靠性

  • 三点估算

    • 考虑估算中的不确定性和风险

    • 三角分布

      • 期望值（平均值） = （最悲观 + 最乐观 + 最可能） / 3

    • 贝塔分布

      • 期望值（平均值） = （最悲观 + 最乐观 + 最可能×4） / 6

      • 标准差（σ）=（最悲观 – 最乐观） / 6

      • 方差（V）= σ^2

      • 平均值±1 σ之间的概率： 68.26%

      • 平均值± 2σ之间的概率： 95.46%

      • 平均值± 3σ之间的概率： 99.73%

        一条路径上A、 B两个紧前紧后活动的均值 = A的均值+B的均值

        一条路径上A、 B两个紧前紧后活动的标准差= A的标准差 ∗ A的标准差 + B的标准差 ∗ B的标准

      • 群体决策技术

      • 储备分析

        • 应急储备、管理储备

# 输出

  • 活动持续时间估算

    不包括提前量和滞后量

# 制定进度计划

定义

分析活动顺序、持续时间、资源需求和进度制约因素，创建进度模型

# 输入

点击查看

项目进度网络图

活动资源需求

资源日历

活动持续时间估算

项目范围说明书

# 工具

点击查看

  • 关键路径法CPM

    • 关键路径

      项目中时间最长的活动顺序，决定着可能的项目最短工期

      可能存在多条关键路径

    • 次关键路径

      时间长度与关键路径最为接近的那条路线

      两者长度越接近，项目风险越大

注意

关键路径的总浮动时间可能是正值、零或负值

关键路径法排出来的进度计划未必可行，关键路径法不考虑资源约束，需要配合资源平衡处理

  • 七格图

  • 顺推找最大

    所有紧前活动的EF中最大的值

  • 逆推找最小

    所有紧后活动的LS中最小的值

    FF <= TF

    EF=ES+DU

    TF=LS-ES=LF-EF

    FF=(所有紧后活动的ES的最小值）- EF

    第一个活动开始时间为0

  • 关键链法CCM

    • 接驳缓冲

    • 项目缓冲

      资源约束型关键路径就是关键链

      是用来应对帕金森（学生综合症）的一种方法

  • 资源优化技术

    • 资源平衡

      减少资源负荷的变化，往往导致关键路径延长

    • 资源平滑

      活动只在其自由和总浮动时间内延迟， 不改变关键路径

  • 进度压缩

    不缩减项目范围的前提下， 缩短工期

  • 赶工

    只适用于那些通过增加资源就能缩短持续时间的，且位于关键路径上的活动

  • 快速跟进

    只适用于相互为选择性依赖关系的活动

:::

# 输出

点击查看

  • 项目进度计划

    • 里程碑进度计划（里程碑图） 向客户汇报
    • 概括性进度计划（横道图、甘特图） 向管理层汇报
    • 详细进度计划（逻辑横道图、时标逻辑图） 优化活动之间的关系

  • 时标逻辑图（时标网络图）

    工期

    最后一个节点对应的标尺刻度

    关键路径

    从第一个节点到最后一个节点所有路径中，没有经过波浪线的路径

    活动持续时间

    活动的开始节点到结束节点之间，横向箭头实线的长度

    活动的自由浮动时间FF

    活动的开始节点到结束节点之间，波浪线的长度

    活动的总浮动时间TF

    从该活动开始节点出发，经过自己到最后节点所有路径中，每条路径上波浪线加总的最小值

# 控制进度

定义

监督项目活动状态， 更新项目进展， 管理进度基准变更

# 输入

项目进度计划

工作绩效数据

# 工具

点击查看
  • 缩短工期的方法
  1. 赶工----投入更多的资源或增加工作时间，以缩短关键活动的工期。
  2. 快速跟进----并行施工，以缩短关键路径的长度。
  3. 使用高素质的资源或经验更丰富的人员。
  4. 甲方同意前提下减小活动范围或降低活动要求
  5. 改进方法或技术，以提高生产效率。
  6. 加强质量管理，及时发现问题，减少返工，从而缩短工期
  • 绩效审查
    • 趋势分析、关键路径法、关键链法、挣值管理、偏差分析

# 输出

  • 工作绩效信息
  • 变更请求
